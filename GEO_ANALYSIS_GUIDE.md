# GEO Analysis Tab — Complete Guide

## What Is GEO Analysis?

GEO stands for **Generative Engine Optimization**. It measures how likely AI-powered search engines (like ChatGPT, Perplexity, Gemini, Grok, and Claude) are to **find, understand, and cite your website** when answering user questions.

The GEO tab is the main output of the tool. It gives you one master score (0–100) plus a detailed breakdown of exactly why you scored that way and what to fix.

---

## Main GEO Tab Layout

When you open the GEO tab, you see three sections stacked vertically:

### 1. Score Hero (top-left)

A large animated ring showing your **AI Citation Readiness Score** from 0–100 with a letter grade (A–F).

- **Green ring** = 80–100 (Excellent)
- **Orange ring** = 65–79 (Good)
- **Red-orange ring** = 50–64 (Fair)
- **Red ring** = below 50 (Poor)

Below the ring are score bars for 8 categories, each with a weight percentage showing how much it contributes to the final score.

**The 8 categories and their weights:**

| Category | Weight | What It Measures |
|---|---|---|
| NLP Intent | 20% | How well your content is structured for AI to extract and quote |
| Structured Data | 20% | Schema markup (JSON-LD, microdata) on your pages |
| E-E-A-T | 15% | Trust signals: About page, author credentials, citations |
| Conversational | 15% | Writing style: FAQ sections, plain language, factual depth |
| Entity | 12% | How well-known your brand is (Wikipedia, LinkedIn, Crunchbase) |
| Probe | 8% | Simulated AI mention rate across 5 engines |
| Technical | 5% | HTTPS, sitemap, broken links, canonical tags |
| Speed & Access | 5% | PageSpeed scores (desktop + mobile) |

**Site Type Badge** — The tool auto-detects what kind of site this is (e-commerce, blog, SaaS, news, portfolio, etc.) and uses that to decide what schema types and signals to recommend.

---

### 2. Engine Scores + Recommendations (top-right)

**Per-Engine Score Cards**

Five cards, one per AI engine, each showing a score and what that engine cares about most:

| Engine | Priorities |
|---|---|
| ChatGPT | Authority, E-E-A-T, comprehensive coverage |
| Perplexity | Freshness, explicit citations, factual claims |
| Gemini | Google signals, Knowledge Graph, technical health |
| Claude | Well-reasoned content, clear structure, conversational depth |
| Grok | Recency, trending topics, real-time relevance |

Each engine re-weights the same 8 category scores using its own priorities — so a site can score well for Claude but poorly for Perplexity.

**Suggestions Panel**

Three tabs — **Critical**, **Important**, **Optional** — listing specific things to fix ranked by how much score they're worth:

- **Critical** = more than 10 points of impact
- **Important** = 5–10 points
- **Optional** = nice-to-have improvements

Each suggestion card is expandable and shows a plain-language "How to fix" section. These are generated by Claude AI using all the analysis data, with a rule-based fallback if the Claude API isn't configured.

### 3. Detail Sub-Tabs (bottom panel)

Seven tabs that dive deep into each analysis area. Each is described below.

## Sub-Tab 1: Schema

**What it shows:** How well your site uses machine-readable structured data (schema markup) so AI engines can understand your content without even reading it.

**Sections inside:**

- **Coverage bar** — What % of your pages have any schema at all (e.g. "8 of 10 pages")
- **Format detection** — Whether you're using JSON-LD (recommended), Microdata, or RDFa
- **Types detected** — Which schema types were found (Article, Product, Organization, FAQPage, BreadcrumbList, etc.)
- **Missing recommended types** — Which types you *should* have based on your site type but don't
- **Completeness issues** — Where you have schema but it's missing required fields (e.g. "Article is missing: datePublished, author")
- **Semantic mismatches** — Where your schema says one thing but the actual page says another (e.g. schema headline doesn't match the H1)

**How the score is computed:**
The backend (`geo_schema.py`) scans every page's HTML for JSON-LD script blocks, Microdata `itemtype` attributes, and RDFa `typeof` attributes. It checks each found schema type against a required-fields list, flags empty or missing fields, and compares schema values (like headline) against actual page content (like H1 text). Coverage = (pages with schema / pages analyzed) × 100.

**Why it matters:** Schema markup is the single biggest lever for AI citation readiness. AI engines use structured data to understand what a page is about without guessing.

---

## Sub-Tab 2: Content

**What it shows:** The depth, structure, and quality of your written content.

**Sections inside:**

- **Avg words per page** — More words generally means more thorough coverage
- **Reading level** — Flesch-Kincaid grade level (Elementary / Middle School / High School / College). AI engines prefer content that's readable but not dumbed-down.
- **FAQ pages** — How many pages have FAQ-style question/answer sections (a huge AI citation signal)
- **Thin content pages** — Pages under 300 words that likely won't get cited

- **Conversational Tone card** — 0–100% score based on use of second-person ("you", "your"), contractions, informal connectors, and questions. Most professional sites score 20–40%.

- **Content Structure card** — % of pages with H2/H3 headings, average headings per page, average number of lists (bullet points)

- **Factual Density card** — 0–100 score counting signals per 1,000 words: statistics (% signs, currency), citations ("according to…"), expert credentials (Dr, PhD), year references, and quoted material

- **FAQ Q&A list** — Expandable list of actual question/answer pairs the tool extracted from your pages

**How the score is computed:**
`geo_content.py` strips navigation, header, and footer HTML then counts words via regex. Flesch-Kincaid grade is calculated from sentence length and syllable count. FAQ detection finds headings that match question patterns and extracts the following paragraph as the answer. Conversational tone is a weighted formula across second-person pronoun density, contraction rate, informal connectors, and question count per 100 words. Factual density uses regex patterns for each signal type, normalized to signals per 1,000 words — 8+ signals/1,000w = a perfect 100.

**Why it matters:** AI engines prefer content with clear structure, factual depth, and direct answers. Thin, vague, or wall-of-text content rarely gets cited.

---

## Sub-Tab 3: E-E-A-T

**What it shows:** How credible and trustworthy your site appears — modeled on Google's E-E-A-T framework (Experience, Expertise, Authoritativeness, Trustworthiness).

**Sections inside:**

- **E-E-A-T score** (0–100) with label: Strong (70+) / Moderate (45–70) / Weak (<45)

- **Trust Pages checklist** — Which of these 6 pages exist on your site:
  - About page
  - Contact page
  - Privacy Policy
  - FAQ page
  - Author/Team pages
  - Case Studies

- **Content Signals checklist** — Whether these 3 things were found:
  - Author bylines (e.g. "Written by Dr. Jane Smith")
  - Research citations (references, sources, DOI links)
  - Content freshness dates (publication/update dates on articles)

- **Expertise signals** — Specific credentials detected in content (e.g. "PhD", "15+ years experience", "award-winning")

- **Content Freshness** — (if a blog is detected) Distribution of posts by age: <30 days, <90 days, <180 days, older — plus blog cadence label: Daily / Weekly / Monthly / Irregular

- **Missing signals** — High-priority items not found (e.g. "No author bylines detected — add author names to every article")

**How the score is computed:**
`geo_eeat.py` checks URL patterns to find trust pages (e.g. `/about`, `/privacy`, `/contact`). It scans body text with regex for byline patterns ("by [Name]", "written by", `itemprop="author"`), citation patterns ("[1]", "according to", DOI links), and credential patterns (Dr, PhD, "years of experience"). For blogs, it extracts dates from `<meta>` tags and article text, buckets them into age groups, and calculates a cadence label from average days between posts.

**Why it matters:** AI engines are less likely to cite a site that has no "About" page, no author names, no dates on articles, and no citations to external sources.

---

## Sub-Tab 4: NLP

**What it shows:** How well your content is structured for AI engines to extract quotes and answer fragments.

**Sections inside:**

- **AI Snippet Readiness** — High / Medium / Low rating. "High" means AI can easily pull a clean answer from your content.

- **Primary Intent** — What your content is mostly about: Informational, Commercial, Transactional, or Navigational. Secondary intents are shown as small badges.

- **Question density** — How many questions appear per 100 words. AI engines look for content that anticipates user questions.

- **Answer blocks** — How many clear answer sections were detected (paragraphs that directly follow a question-pattern heading)

- **Synonym richness** — High / Medium / Low. Whether you use varied vocabulary or repeat the same words constantly.

- **Answer Quality card** — 4 metrics on how well your answers are written:
  - **BLUF format** — Does the answer start with the direct answer (no preamble)? Ideal: 60%+
  - **Avg length** — Ideal answer length for AI citation is 40–120 words
  - **Self-contained** — Can the answer be understood without reading the rest of the page? Ideal: 60%+
  - **Confident tone** — Low use of hedge words like "might", "could", "possibly". Ideal: 70%+

- **Key Topics** — Badge list of patterns detected: How-to, What-is, Comparison, Best, FAQ

- **AI Analysis box** — A plain-language explanation generated by Claude about the site's NLP readiness (or a rule-based summary if Claude API isn't configured)

**How the score is computed:**
`geo_nlp.py` uses the Claude API (if configured) for semantic analysis — asking Claude to assess structure, question coverage, answer quality, and intent clarity. Without the API, it falls back to rule-based heuristics: question density from regex count / word count × 100, BLUF detection by checking if the first sentence is under 30 words, self-contained check by looking for context-referencing pronouns at the start of answers, and confident tone by measuring hedge-word density.

**Why it matters:** Even if your content is well-written, AI engines need it to be structured so extracting clear, accurate answers is easy. BLUF format and direct answers are the biggest wins here.

## Sub-Tab 5: Visibility

**What it shows:** A simulated test of whether AI engines would actually mention your site when answering relevant questions.

**How it works:**
1. Claude generates 3–5 questions where your site would be a natural answer
2. Claude then simulates how each of the 5 AI engines would respond (acting as each engine in turn using engine-specific system prompts)
3. The tool checks whether your domain appears in those simulated responses
4. It calculates a mention rate per engine

**Sections inside:**

- **Overall mention rate** — 0–100% across all engines. Label: High (70%+) / Medium (50–70%) / Low (25–50%) / Not Visible (<25%)

- **Engine cards** — One per engine showing their individual mention rate and how many times your domain was mentioned out of total queries

- **Query results table** — Each generated question as a row. Clicking expands it to show what each engine "said" and whether your domain was mentioned, with the actual response excerpt shown.

**Important note:** This is *simulated* visibility — Claude mimics other engines' response styles; it doesn't actually query those engines. Treat it as a directional signal, not a definitive measurement.

**How the score is computed:**
`geo_probe.py` requires the ANTHROPIC_API_KEY. It first calls Claude to generate brand-relevant questions, then calls Claude 5 more times — once per engine — with a system prompt that instructs it to respond as that engine would. It checks if the site's domain appears in each response and computes: `mention_rate = (mentions / total_queries) × 100` per engine. If the API key is missing, a neutral score of 50 is returned so the missing data doesn't unfairly penalize the overall score.

**Why it matters:** This is the most direct measure of whether all your other GEO work is paying off. A site can have great schema and E-E-A-T but still fail to appear in AI responses if it's not well-known enough.

## Sub-Tab 6: Entity

**What it shows:** How well-established your brand is as a recognized entity in the broader information ecosystem — a critical factor for AI citation.

**Sections inside:**

- **Entity score** (0–100) with label: Established / Emerging / Unknown

- **Score breakdown bars (4 categories):**

  | Category | Max Points | What Earns Points |
  |---|---|---|
  | Wikipedia | 35 pts | A Wikipedia article exists for your brand |
  | sameAs Profiles | 30 pts | Links to LinkedIn, Crunchbase, Wikidata, Twitter, etc. in your schema |
  | Organisation Schema | 20 pts | How complete your Organization JSON-LD schema is (8 fields) |
  | Authority Links | 15 pts | Outbound links to .gov, .edu, Wikipedia, major news sites |

- **Wikipedia card** — Whether a Wikipedia article was found for your brand. If found, links to it. If not, explains that this is the highest single-impact action you can take.

- **sameAs Profiles card** — Which profiles were found and their point values:
  - Wikipedia (15 pts), Wikidata (12 pts), LinkedIn (5 pts), Crunchbase (4 pts), Twitter/X (3 pts), and others

- **Organisation Schema card** — Checklist of 8 fields checked in your Organization JSON-LD: name, url, logo, description, sameAs, address, telephone, foundingDate

- **Authority Links card** — Which high-authority domains you link out to from your pages (wikipedia.org, .gov, .edu, Crunchbase, major news outlets)

**How the score is computed:**
`geo_entity.py` looks up your brand name (extracted from schema, og:site_name, page title, or domain) in the Wikipedia API. It parses your Organization schema's `sameAs` array and scores each platform by its authority weight. It counts how many of the 8 org schema fields are filled in. It extracts all outbound `<a href>` targets from your pages and scores known authority domains. Entity score = (Wikipedia pts + sameAs pts + Org schema pts + Authority pts) / 100.

**Why it matters:** AI engines are more likely to cite brands they can verify through cross-referenced sources. Wikipedia is the gold standard, but LinkedIn + Crunchbase + solid org schema moves you from "Unknown" to "Emerging".

---

## Sub-Tab 7: Pages

**What it shows:** Individual GEO scores for every page that was analyzed, sorted worst-first so you can prioritize fixes.

**Sections inside:**

- **Summary bar** — "X pages scored · Y need attention" (pages with scores under 65)

- **Page rows** — Each analyzed page showing:
  - Score (0–100) + grade badge (A–F)
  - URL path (shortened to 55 characters)
  - Issue count badges ("2 critical", "1 important")
  - Per-engine mini scores: Claude, ChatGPT, Gemini, Grok, Perplexity

- **Expanded row (click any row)** — Shows:
  - Signal badges: Author detected? Date detected? Citations found? Reading grade?
  - 5-category breakdown bars with exact scores:
    - Structured Data (25% weight)
    - E-E-A-T (25% weight)
    - Content (20% weight)
    - Meta completeness — title, H1, description, canonical (15% weight)
    - NLP / Semantic quality (15% weight)
  - Issues list with Critical / Important / Optional priority labels and specific messages

**How each page is scored:**

| Category | What earns a high score |
|---|---|
| Structured Data | Has JSON-LD schema with all required fields filled |
| E-E-A-T | Has author byline, publication date, citation patterns detected |
| Content | 1000+ words, uses H2/H3 headings, has FAQ section, lists present |
| Meta | Has title, H1, meta description, and canonical tag (1 pt each) |
| NLP | Conversational tone, question/answer pairs, how-to/what-is patterns |

Per-engine scores on each page use the same 5 raw category scores re-weighted by each engine's priority profile.

**Why it matters:** The site-level score averages across all pages, but your worst pages drag it down. This tab shows exactly which pages to fix first and what's wrong with each one.

---

## How the Final Score Is Calculated

1. Each analyzer runs and produces a raw 0–100 score for its category
2. Each raw score is multiplied by its weight (e.g. NLP raw score × 0.20)
3. All weighted scores are added together
4. The result is the AI Citation Readiness Score (0–100)
5. Grade is assigned: A ≥ 90, B ≥ 80, C ≥ 65, D ≥ 50, F < 50

For per-engine scores, the exact same raw scores are re-weighted using each engine's own priority percentages, producing a separate score per engine.

## Backend Files Reference

| File | What It Does |
|---|---|
| `geo_site_type.py` | Detects site type (blog, e-commerce, SaaS, etc.) from URL patterns, schema, and keywords |
| `geo_schema.py` | Extracts and validates structured data (JSON-LD, Microdata, RDFa) |
| `geo_content.py` | Measures word count, reading level, FAQ presence, tone, factual density |
| `geo_eeat.py` | Checks trust pages, author signals, citations, expertise credentials, freshness |
| `geo_nlp.py` | Evaluates snippet readiness, intent, question density, answer quality (uses Claude API) |
| `geo_entity.py` | Checks Wikipedia, sameAs profiles, org schema completeness, authority links |
| `geo_probe.py` | Simulates AI engine mention rate (uses Claude API × 5 engines) |
| `geo_page_scores.py` | Scores every individual page across 5 categories + per-engine breakdown |
| `geo_score.py` | Aggregates all category scores into the final unified score |
| `geo_suggestions.py` | Generates prioritized fix recommendations (uses Claude API with rule-based fallback) |

## Quick Reference: What to Fix First

| Score Range | Priority Actions |
|---|---|
| < 50 (F) | Add Organization schema, create About/Contact pages, add author bylines, fix missing H1/title/meta description |
| 50–64 (D) | Add Article/Product schema, start a blog with dated posts, add FAQ sections to key pages |
| 65–79 (C) | Improve answer quality (BLUF format), add sameAs profile links, build authority outbound links |
| 80+ (B/A) | Fine-tune entity signals, pursue Wikipedia article, boost weak per-engine scores |

## API Endpoints

All GEO data is served from these backend routes (authentication required):

```
GET /sites/{task_id}/geo              → Full results (all sub-tabs)
GET /sites/{task_id}/geo/score        → Score + breakdown only
GET /sites/{task_id}/geo/schema       → Schema sub-tab data
GET /sites/{task_id}/geo/content      → Content sub-tab data
GET /sites/{task_id}/geo/nlp          → NLP sub-tab data
GET /sites/{task_id}/geo/eeat         → E-E-A-T sub-tab data
GET /sites/{task_id}/geo/entity       → Entity sub-tab data
GET /sites/{task_id}/geo/probe        → Visibility sub-tab data
GET /sites/{task_id}/geo/page-scores  → Pages sub-tab data
GET /sites/{task_id}/geo/suggestions  → Suggestions panel data
GET /sites/{task_id}/geo/site-type    → Detected site type
GET /sites/{task_id}/geo/export       → CSV or PDF export (Agency plan)
```
