## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight label/input spacing |
| sm | 8px | Compact element spacing (gap-2 in existing pages) |
| md | 16px | Default element spacing, card internal padding (p-6 = 24px) |
| lg | 24px | Card padding (`p-6`), section breaks within cards |
| xl | 32px | Gap between plan cards on desktop grid |
| 2xl | 48px | Vertical breathing room above/below card grid |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Touch targets for CTA buttons inside plan cards must be minimum 44px tall (accessibility requirement — existing auth buttons use `py-2` which renders ~28px; plan CTAs must explicitly set `min-h-[44px]`).

## Typography

Existing project uses `text-xs` (12px) for labels, form body, and button text. Auth pages set the heading at `text-sm` (14px). Plan cards introduce a display tier for pricing amounts.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / label | 12px (text-xs) | 400 (normal) | 1.5 |
| Heading | 14px (text-sm) | 600 (semibold) | 1.3 |
| Plan name | 16px (text-base) | 600 (semibold) | 1.2 |
| Price display | 28px (text-3xl) | 600 (semibold) | 1.1 |

Font weights in use: 400 (normal) and 600 (semibold) only. The 28px size at semibold provides sufficient visual hierarchy for pricing numbers without introducing a third weight tier.

Source: inherited from `signup/page.tsx` and `login/page.tsx` patterns. Price display tier is new for this phase — no existing precedent.

## Component Inventory

### 1. `/select-plan` page — Plan Selection Screen

**Layout:** Full-page centered layout (`flex min-h-screen items-center justify-center`), matching the existing `/signup` and `/login` page shell. No sidebar, no app chrome.

**Header block:** Same brand header used on auth pages — link icon SVG + "AI SEO TOOL" wordmark in `var(--accent)`, `text-sm font-bold`. Below the wordmark: heading "Choose your plan" (`text-sm font-semibold`) and subtext "Select the plan that fits your needs. You can upgrade at any time." (`text-xs var(--muted)`).

**Plan card grid:** Three cards arranged in a horizontal row on screens >= 768px; stacked vertically on mobile. Cards use `var(--surface)` background with `1px solid var(--border)` border and `rounded-lg`. The Pro card has an additional `2px solid var(--accent)` border to signal recommendation.

**Each plan card contains (top to bottom):**
- Plan name: `text-base font-semibold var(--foreground)`
- Optional badge (Pro only): "Most Popular" chip — `text-xs font-semibold` pill in `var(--warning)` text on `#fefce8` background
- Price display: monthly price or "Free" — `text-3xl font-semibold var(--foreground)`, subscript `/month` in `text-xs var(--muted)`
- One-line description: `text-xs var(--muted)`
- Feature list: bulleted list items with a checkmark SVG icon (14x14, `var(--accent)` for Pro/Agency, `var(--muted)` for Free) and `text-xs var(--foreground)` label
- CTA button: full-width, `min-h-[44px]`, `rounded`, `text-xs font-semibold`, `text-white`. Accent background (`var(--accent)`) for Pro. Secondary style (`var(--surface-elevated)` bg, `1px solid var(--border)`, `var(--foreground)` text) for Free and Agency.

**Hover / focus states:**
- Card: on hover, border transitions to `var(--border-dark)`. Pro card: border stays `var(--accent)`.
- CTA button: `var(--accent-hover)` (#14532d) on hover for accent buttons. `var(--border-dark)` border on hover for secondary-style buttons.
- Focus: `focus:outline-none focus:ring-1 focus:ring-[var(--accent)]` on all interactive elements (matching existing auth patterns).

**Loading state (after CTA click):** Button text changes to context-appropriate loading label (see Copywriting). Button disabled (`disabled:opacity-50 disabled:cursor-not-allowed`). All three cards show `pointer-events-none` while redirect is in flight to prevent double-click.

### 2. Upgrade Modal — Quota Exceeded

Triggered when a 402 response is received from the `/analyze/` endpoint.

**Shell:** Matches `SessionExpiredModal.tsx` exactly — `fixed inset-0 z-50 flex items-center justify-center`, black 60% overlay (`rgba(0,0,0,0.6)`), inner card `w-full max-w-sm rounded-lg p-6 var(--surface) 1px solid var(--border)`.

**Content (top to bottom):**
- Heading: "Audit limit reached" — `text-sm font-semibold var(--foreground)`
- Body: Two lines. Line 1: current plan + quota explanation (see Copywriting). Line 2: upgrade CTA text. — `text-xs var(--muted) mt-1 mb-4`
- Single CTA button: "Upgrade Plan" — full-width, `min-h-[44px]`, accent background, same style as primary auth button. Clicking navigates to `/select-plan`.
- Secondary dismiss link: "Not now" — `text-xs var(--muted)` centered below button, no underline, hover `var(--foreground)`.

**Accessibility:** `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading id. Focus trapped inside modal while open. Escape key closes the modal (sets open=false).

## Interaction Contracts

### `/select-plan` Route Guard

- Authenticated users with no subscription record: always land on `/select-plan` (enforced by `proxy.ts` route guard — D-13).
- Authenticated users with an existing subscription: redirect to `/` immediately (guard must check subscription presence alongside auth state).
- Unauthenticated users: redirect to `/login` (existing guard behavior unchanged).
- The `/select-plan` route must be listed in `proxy.ts` as auth-required but pre-subscription (it must not redirect to itself in a loop).

### Free Plan Selection Flow

1. User clicks "Get Started Free".
2. Button enters loading state: label changes to "Setting up your account...".
3. Frontend calls backend `POST /subscriptions/select` with `{ plan: "free" }`.
4. On 200 success: `window.location.href = "/"` — hard redirect to app.
5. On error: inline error message below the card grid (not inside the card), `text-xs var(--error)`, `role="alert"`.

### Pro / Agency Plan Selection Flow

1. User clicks "Start Pro" or "Start Agency".
2. Button enters loading state: "Redirecting to checkout...".
3. Frontend calls backend `POST /subscriptions/checkout` with `{ plan: "pro" | "agency" }`.
4. Backend returns `{ checkout_url: "https://checkout.stripe.com/..." }`.
5. Frontend performs `window.location.href = checkout_url` — full browser redirect to Stripe Checkout hosted page.
6. On Stripe success: Stripe redirects to `success_url` (e.g. `/select-plan/success?session_id={CHECKOUT_SESSION_ID}`). Frontend displays a brief "Payment confirmed — loading your account..." message while the backend webhook activates the subscription, then polls `/auth/me` for updated subscription state before redirecting to `/`.
7. On Stripe cancel: Stripe redirects to `cancel_url` = `/select-plan`. User is back on the plan selection screen with no error shown (they voluntarily cancelled).
8. On backend error (non-2xx): inline error below card grid, `text-xs var(--error)`, `role="alert"`. Error copy: see Copywriting section.

### Upgrade Modal Trigger

1. `apiFetch` receives a 402 response from any endpoint.
2. A `quota:exceeded` custom event is dispatched on `window` (mirrors `auth:expired` pattern from SessionExpiredModal).
3. A `UpgradeModal` component (mounted in `layout.tsx` alongside `SessionExpiredModal`) listens for `quota:exceeded` and sets `open=true`.
4. Modal presents upgrade copy and "Upgrade Plan" button.
5. "Upgrade Plan" navigates to `/select-plan` and closes the modal.
6. "Not now" closes the modal with no navigation.
7. Escape key closes the modal with no navigation.
