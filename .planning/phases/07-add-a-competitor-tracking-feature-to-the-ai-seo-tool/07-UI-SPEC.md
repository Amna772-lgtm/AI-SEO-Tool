## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox-label spacing, inline badge padding |
| sm | 8px | Within-card element spacing, button padding vertical |
| md | 16px | Default card padding, section gaps, input padding |
| lg | 24px | Between competitor cards in the grid row, panel section gaps |
| xl | 32px | Between major layout zones (selector bar → competitor row → comparison view) |
| 2xl | 48px | Tab content top padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Competitor suggestion card checkbox touch target: 24px × 24px minimum (accessibility requirement; nearest 4px-grid value above 20px)
- Tab bar height matches existing nav: 40px (consistent with other tabs in page.tsx)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 |
| Heading | 16px (text-base) | 600 (semibold) | 1.3 |
| Display | 20px (text-xl) | 600 (semibold) | 1.2 |

Source: inferred from existing components (GeoScoreRing uses `fontSize={size * 0.24}` at size=140 → ~33px for score number; EngineScores uses `text-sm font-semibold`, `text-xs`, `text-lg font-bold`; page.tsx uses `text-xs font-semibold uppercase tracking-wide` for section headers). This phase uses the same four-size system with a maximum of two weights (400 and 600). Weight 700/bold is not used in this phase; 600/semibold serves as the heaviest weight throughout.

Monospace exception: competitor domain names in suggestion cards use `font-mono text-xs` — consistent with URL display elsewhere (broken links list, crawl table address column).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #f5f6f8 (var(--background)) | Page background, tab content area |
| Secondary (30%) | #ffffff (var(--surface)) + #eef0f3 (var(--surface-elevated)) | Competitor cards, suggestion cards, comparison panels, card headers |
| Accent (10%) | #166534 (var(--accent)) | "Find Competitors" CTA button, "Add Competitors" confirm button, active checkbox fill, upgrade button in LockedFeature |
| Destructive | #dc2626 (var(--error)) | Remove competitor button, delete confirmation |

Accent reserved for:
1. "Find Competitors" primary button (the single call-to-action that triggers the Claude discovery)
2. "Add Competitors" confirm button (after selection on suggestion cards)
3. "Re-audit" refresh button on each competitor card (secondary/outline variant using accent border + accent text)
4. Checked checkbox fill on suggestion cards
5. "Upgrade to Pro" button in the LockedFeature upgrade prompt (inherits from existing LockedFeature component)

Secondary semantic color — warning (#ca8a04 via var(--warning)):
- Used on the lock icon in LockedFeature (already established pattern — do not change)
- Used on "Auditing..." pending badge on competitor cards while analysis is in progress

Score color bands (inherited from existing components — do not change):
- Score >= 80: #16a34a (emerald)
- Score >= 65: #ca8a04 (amber)
- Score >= 50: #ea580c (orange)
- Score < 50: #dc2626 (red)

**Primary focal point:** GeoScoreRing score on the primary site card; secondary focal point: radar chart polygon comparison.

## Component Inventory

All new components live in `frontend/app/components/competitors/`.

### CompetitorsTab.tsx
Top-level tab container. Layout: flex column, full height within tab panel.

Sections (top to bottom):
1. Primary site selector bar (full width, height 56px, bg var(--surface), border-b)
2. Competitor row (horizontal scroll on small viewports, flex row with gap-lg)
3. Comparison view (full width below competitor row)

### PrimarySiteSelector.tsx
Dropdown `<select>` populated from history API. Label: "Your site". Width: 320px max. Inline with "Find Competitors" button. Shows placeholder "Select an audited site..." when no site chosen.

### CompetitorSuggestionCard.tsx
Card shown during the discovery confirmation step. Dimensions: min-width 240px, border rounded-lg, bg var(--surface), border var(--border). Contents:
- Top-right: checkbox (24×24px minimum touch target, accent fill when checked)
- Headline: domain in font-mono text-sm font-semibold text-[var(--foreground)]
- Body: one-line reason in text-xs text-[var(--muted)], max 2 lines, ellipsis overflow

### CompetitorCard.tsx
Card for an already-queued or completed competitor. States: pending (spinner + "Auditing..." label), completed (GeoScoreRing + grade + site type label), error (red border + error message). Width: 200px. Actions row: "Re-audit" button (accent outline), "Remove" button (destructive text-only).

### CompetitorRadarChart.tsx
Custom SVG polygon chart. No external library. 6 axes: NLP, Schema, E-E-A-T, Content, Entity, Technical. One colored polygon per site. Primary site: #4f46e5 (indigo, consistent with ScoreTrendChart "overall" series color). Competitors: cycling palette of #2563eb, #7c3aed, #d97706, #0891b2, #db2777 (same series colors as ScoreTrendChart). Polygon fill opacity: 0.15. Polygon stroke width: 2px. Axis labels: text-xs text-[var(--muted)]. Legend below chart: one colored dot + domain per site. Chart size: 400×400px SVG viewBox, responsive via width="100%".

### SiteComparisonCard.tsx
Side-by-side card used in the comparison section. Reuses GeoScoreRing (size=100) and EngineScores component directly. Card width: flex-1 min-width 220px. Layout: GeoScoreRing centered, domain label below (font-mono text-xs), grade badge, site type badge (text-[10px] bg-[var(--surface-elevated)] rounded px-1.5 py-0.5).

## Interaction States

### Tab — Competitors (plan-gated)
- Free user: tab label visible, clicking renders `<LockedFeature title="Competitor Tracking" plan="Pro" />` centered in tab content. No other content visible.
- Pro/Agency user: full feature UI visible.

### Primary Site Selector — No History
- Dropdown shows single disabled option: "No audited sites yet — run an audit first"
- "Find Competitors" button is disabled (opacity-50 cursor-not-allowed)

### Primary Site Selector — Site Selected, No Competitors
- "Find Competitors" button active (accent bg, white text)
- Below selector: empty state (see Copywriting Contract)

### Find Competitors — Loading
- Button replaces text with inline spinner + "Finding competitors..." label
- Button disabled while loading
- Spinner: 12×12px, border-2 border-[var(--border)] border-t-[var(--accent)], animate-spin (consistent with `Spinner` component in page.tsx)

### Find Competitors — Suggestions Returned
- Discovery section appears below selector bar
- Heading: "Found N competitors" (text-sm font-semibold)
- Suggestion cards in flex-wrap row with gap-md
- Manual add input: text input + "Add URL" button inline (same row, below suggestion cards)
- "Add X Competitors" button (accent, disabled if no cards checked)
- Count badge shows "(X selected)" next to button

### Find Competitors — Claude Failure / No Suggestions
- Discovery section shows fallback message (see Copywriting Contract, error state)
- Manual add input still available

### Competitor Card — Pending
- Spinner centered in card, label "Auditing..."
- Card border: var(--border), normal weight
- Re-audit and Remove buttons hidden while pending

### Competitor Card — Complete
- GeoScoreRing (size=100), domain, grade, site type
- Re-audit button (outline, accent color): visible
- Remove button (text-only, var(--error)): visible
- Clicking card header domain opens history detail for that analysis

### Competitor Card — Error
- Border: #dc2626 border-red-500
- Body: "Audit failed" in text-xs text-red-500
- Sub-copy: "Check the URL and try again"
- Re-audit button visible (accent outline)

### Remove Competitor — Confirmation
- Inline confirmation within the card (not modal): card expands, shows "Remove [domain]? This cannot be undone." with "Keep Competitor" (text button) and "Remove" (red filled button). No separate modal.

### Radar Chart — Hover
- Hovering an axis label highlights the corresponding polygon with fill opacity 0.35 (from 0.15)
- Tooltip: absolute positioned div showing site domain + dimension score, bg var(--surface), border var(--border), text-xs, shadow-sm

## Accessibility Notes

- Competitor suggestion checkboxes must have explicit `<label>` elements wrapping the card or using `aria-label` — not click handlers on a `div`.
- "Find Competitors" button must be a `<button type="button">` with meaningful aria-label when in loading state: `aria-label="Finding competitors, please wait"`.
- Radar chart SVG must include `role="img"` and `aria-label="GEO dimension comparison chart"` with a `<title>` element.
- Score color alone does not convey state — grade letter (A/B/C/D/F) always accompanies the color-coded ring.
- Remove confirmation must be focusable inline — not hidden from keyboard navigation.
