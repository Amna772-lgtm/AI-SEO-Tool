## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (two separate contexts — see note) | RESEARCH.md |

## Spacing Scale

### WordPress Plugin (wp-admin context)

WordPress admin uses an 8-point base grid. The plugin uses `@wordpress/components` spacing utilities which map to this scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline gaps |
| sm | 8px | Control spacing within form groups, compact list item gaps |
| md | 16px | Card internal padding, between controls, section body padding |
| lg | 24px | Between card sections, between the progress bar and result content |
| xl | 32px | Between major screen sections (e.g., quota card and history list) |
| 2xl | 48px | Top/bottom page breathing room |
| 3xl | 64px | Not used in WP plugin (WP admin chrome consumes vertical space) |

Exceptions:
- Connection screen center card: `max-width: 480px`, centered with `auto` horizontal margins — WP admin does not use full-bleed layout for onboarding flows
- Progress bar height: 8px (fixed, not on spacing scale — follows existing main tool `progress-indeterminate` pattern)
- Touch targets on WP admin buttons: min-height 32px (WP admin standard, enforced by `@wordpress/components` Button)

### Next.js API Keys Page (existing dashboard context)

Inherits the existing 8-point scale already in use across the dashboard. No new spacing tokens defined. All padding/margin values use Tailwind utility classes referencing the existing scale (p-4 = 16px, p-6 = 24px, gap-4 = 16px).

Exception: `gap-3 = 12px` — Tailwind v4 default gap for tight list items; used in key list row internal gap only. Approved exception — existing dashboard pattern consistent with other list rows in the codebase.

## Typography

### WordPress Plugin

Typography is inherited from WordPress admin — the plugin does NOT declare custom font sizes. The following four sizes represent all declared type sizes in the plugin. Maximum 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Small labels / links | 12px | 400 | 1.5 | Engine name, focus description on EngineScore cards; "View Full Report" link; "Sign up" link below connection card; grade badge text |
| Body / description / table cells | 13px | 400 | 1.5 | WP admin default body; card descriptions, table cells, phase label during analysis |
| Labels / section headings | 13px | 600 | 1.3 | Form labels, card section titles (`<h3>` inside WP cards), score number on EngineScore cards |
| Score number (custom SVG) | 28px | 600 | 1.0 | Used inside GeoScoreRing SVG center text and quota count display |

**Note on 23px screen heading:** The WP admin `<h1 class="wp-heading-inline">` renders at ~23px via the WP admin stylesheet. This is a WP-admin-inherited value — it is NOT a declared plugin type size and does not count toward the 4-size limit.

Font family: inherits `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen-Sans, Ubuntu, Cantarell, sans-serif` from WP admin. Do NOT declare `font-family` in plugin CSS — let WP admin cascade apply it.

### Next.js API Keys Page

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label / small heading | 13px | 600 | 1.3 |
| Section heading | 16px | 600 | 1.2 |
| Display / key value | 13px (monospace) | 400 | 1.5 |

Font family: `var(--font-inter), system-ui, sans-serif` for all text. Monospace font `var(--font-geist-mono)` for the raw API key display field (one-time reveal).

## Color

### WordPress Plugin

The plugin inherits WordPress admin colors and does NOT override them. The following values describe how existing WP admin tokens map to the plugin's surfaces:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f0f0f1` (WP admin page background) | Main plugin page background — inherited, not declared |
| Secondary (30%) | `#ffffff` (WP admin card/metabox background) | Card surfaces, `@wordpress/components` Card background |
| Accent (10%) | `#0d9488` (main tool teal — `--accent`) | Score ring arc color, "Connect Account" button, "Analyze" button, active progress segment |
| Success state | `#16a34a` | Score ≥ 80 ring/text color (matches GeoScoreRing color logic exactly) |
| Warning state | `#ca8a04` | Score 65–79 ring/text color; quota approaching limit |
| Error / destructive | `#dc2626` | Score < 50 ring/text color; error notices; revoke key confirmation |
| Plan gate lock | `#d97706` | Lock icon and "Upgrade" prompt (matches existing `--warning` token) |
| WP admin border | `#c3c4c7` (WP admin standard) | Card borders, table borders — use `@wordpress/components` defaults, not custom CSS |

**Accent `#0d9488` is reserved for:**
- "Connect Account" button (ConnectionScreen)
- "Analyze This Site" button (DashboardScreen)
- GeoScoreRing arc when score is not yet categorized (loading state)
- Active phase segment in the progress bar
- Active tab underline indicator (WP `TabPanel` active state overridden to teal)

**Do NOT apply accent color to:** table row hover states, link text, section headings, or any non-interactive decorative element.

### Next.js API Keys Page

Inherits the full existing CSS custom property palette from `globals.css`. No new color tokens are added.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` = `#f8fafc` | Page background |
| Secondary (30%) | `var(--surface)` = `#ffffff` | Key cards, form card |
| Accent (10%) | `var(--accent)` = `#0d9488` | "Generate API Key" button; copy-to-clipboard icon active state |
| Destructive | `var(--error)` = `#dc2626` | "Revoke Key" button; revoke confirmation text |

**Accent `var(--accent)` is reserved for:**
- "Generate API Key" primary CTA button
- Copy icon hover/active state on the one-time key reveal

## Interaction States

### Connection Screen (WordPress plugin)

| State | Component | Behavior |
|-------|-----------|---------|
| Idle | API key TextControl + Connect Account button | Button enabled only when input is non-empty (>0 chars after trim) |
| Validating | Connect Account button | Button shows `<Spinner />` inline, disabled, label changes to "Connecting…" |
| Error — invalid key | Notice below form | `<Notice status="error">` — "Invalid API key. Check your key in AI SEO Tool → Settings → API Keys." |
| Error — network | Notice below form | `<Notice status="error">` — "Could not reach the AI SEO Tool backend. Check the backend URL in Settings." |
| Success | Full screen transition | Connection screen unmounts; DashboardScreen mounts. No page reload. |

### Dashboard Screen (WordPress plugin)

| State | Component | Behavior |
|-------|-----------|---------|
| Idle (no prior analysis) | Analyze button | Full-width primary Button with teal background. Disabled if quota exhausted. |
| Analyzing | Progress bar | `AnalysisProgress` component replaces the Analyze button area. Three labeled segments: "Crawling" → "Technical Checks" → "GEO Analysis". Each segment fills sequentially based on job `status` field. Bar uses `#0d9488` for completed, `#c3c4c7` for pending. |
| Analysis complete | Result tabs | Tab content replaces progress bar. DashboardTab is shown by default. |
| Analysis failed | Error notice | `<Notice status="error">` with job error message + "Try Again" button. |
| Quota exhausted | Analyze button | Button disabled, `<Notice status="warning">` — "You've used all {N} audits this month. Upgrade to run more." with "Upgrade" link to main tool. |

### Plan Gate (WordPress plugin — Free plan)

- GEO Analysis tab, Technical Audit tab, and History trend chart are replaced by a `PlanGate` component.
- `PlanGate` renders: lock icon (Dashicons `dashicons-lock`, 28px, `#d97706`), heading "Upgrade to Pro", body "This feature is available on the Pro plan.", `<Button variant="primary">` labeled "Upgrade Plan" that links to the main tool's pricing page in a new tab.
- The WordPress admin sub-menu items (GEO Analysis, Technical Audit, History) remain visible for free users — clicking them shows the plan gate, not a 404.

### Progress Bar Phasing

Three phases map to job `status` values as follows:

| Phase Label | Triggered When | Bar Fill |
|-------------|---------------|---------|
| Crawling | status = `crawling` or `pending` | 33% |
| Technical Checks | status = `technical` | 66% |
| GEO Analysis | status = `geo` | 90% |
| Complete | status = `completed` | 100% (bar hidden, results shown) |

Progress bar height: 8px. Border-radius: 4px. Background track: `#c3c4c7`. Fill color: `#0d9488`. Transition: `width 0.4s ease`. Phase label displayed above the bar in 13px, weight 600.

### "View Full Report" Link

Each result tab (Dashboard, GEO Analysis, Technical Audit, History) shows a right-aligned "View Full Report →" link in the tab header bar. This is a plain `<a>` tag with `target="_blank"`. Style: 12px, `var(--muted)` gray, no underline by default, underline on hover. The link constructs the URL as: `{main_tool_url}/dashboard?site={site_id}` using the `aiSeoTool.mainAppUrl` localized variable.

### WordPress Plugin Components

All components use `@wordpress/components` as the primitive layer. No custom CSS framework.

| Component | WP Primitives Used | Notes |

| `DashboardScreen` | `Card`, `CardHeader`, `CardBody`, `Button`, `TabPanel` | Top card: plan name, quota used/limit, last audit date. TabPanel for result tabs. |

| `DashboardTab` | `Card`, custom SVG (GeoScoreRing), table via `<table>` HTML | GeoScoreRing is a pure SVG — re-implement with same radius/stroke/color logic as existing `GeoScoreRing.tsx`. |

### GeoScoreRing Re-implementation (WordPress plugin)

The WordPress plugin re-implements `GeoScoreRing` as a standalone SVG component (no external dependency). Use identical math to `frontend/app/components/geo/GeoScoreRing.tsx`:

- Default size: 120px (smaller than main app's 140px — WP admin sidebar takes horizontal space)
- Stroke width: 10px
- Color thresholds: ≥80 `#16a34a`, ≥65 `#ca8a04`, ≥50 `#ea580c`, <50 `#dc2626`
- Center text: score number at `fontSize = size * 0.24`, weight 600
- Sub-text: "/ 100" at `fontSize = size * 0.1`, color `#757575` (WP admin muted gray)
- Grade badge: 32px circle below ring, same color as score arc, white text, weight 600

### EngineScore Cards (WordPress plugin)

Re-implement `EngineCard` using `@wordpress/components` Card as wrapper. Each card:
- Engine name: 12px, weight 400, color `#757575`
- Score number: 13px, weight 600, color from threshold function
- Progress bar: 6px height custom div (same pattern as DashboardTab)
- Focus description: 12px, weight 400, `#757575`
- Grid: 2 columns on narrow WP admin, 5 columns when admin menu is collapsed (use CSS `grid-template-columns: repeat(auto-fill, minmax(140px, 1fr))`)

## Accessibility Contract

### WordPress Plugin

- All `TextControl` inputs have explicit `label` prop (not aria-label alone)
- All `Button` elements have visible text labels — no icon-only buttons without `aria-label`
- `Notice` components use `status="error"` / `status="warning"` / `status="success"` so WP renders the correct ARIA role
- Progress bar: `<div role="progressbar" aria-valuenow={percent} aria-valuemin={0} aria-valuemax={100} aria-label="Analysis progress">`
- Tables use `<th scope="col">` headers for all column headers
- `TabPanel` from `@wordpress/components` is ARIA-compliant by default — do not replace with custom tab implementation

### Next.js API Keys Page

- Key name TextInput: `htmlFor` label linked to input `id`
- "Copy Key" button: `aria-label="Copy API key to clipboard"`
- Revoke confirmation: rendered as a `<dialog>` element or modal using existing pattern from `UpgradeModal.tsx`
- Monospace key display: `<code>` element with `aria-label="Your new API key"`
