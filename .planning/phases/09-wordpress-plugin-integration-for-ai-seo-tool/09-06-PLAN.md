**Compliance checklist to verify and fix:**

1. **GPL Headers (D-19)**: Every PHP file must have:
```php
/**
 * @package AI_SEO_Tool
 * @license GPL-2.0-or-later
 */
```
Check all 4 PHP files. Add if missing.

2. **i18n audit**: Search all `.jsx` files for any user-facing string NOT wrapped in `__()` from `@wordpress/i18n`. Fix any missed strings. Key patterns to check:
   - Button labels
   - Notice messages
   - Tab labels in TabPanel
   - Table column headers
   - Status text
   - Error messages

3. **Sanitization audit**: In `class-rest-proxy.php`:
   - All `$_GET` access uses `sanitize_key()`
   - All `get_param()` values from REST requests use `sanitize_text_field()` before use
   - All `update_option()` calls: API key uses `sanitize_text_field()`, backend URL uses `esc_url_raw()`
   - No direct `$_POST` or `$_REQUEST` access (all through WP REST API)

4. **Activation hook audit**: Verify `class-activator.php` contains ONLY `add_option()` calls. No `wp_remote_get`, `wp_remote_post`, `file_get_contents`, `curl_*`, or any HTTP function.

5. **readme.txt completeness**: Ensure all required sections per WordPress.org:
   - `=== Plugin Name ===` header with Contributors, Tags, Requires at least, Tested up to, Stable tag, Requires PHP, License, License URI
   - `== Description ==` section
   - `== Installation ==` section (numbered steps)
   - `== Frequently Asked Questions ==` section
   - `== Changelog ==` section
   - `== Screenshots ==` section (can be empty with placeholder)

6. **Text Domain consistency**: Verify `Text Domain: ai-seo-tool` in plugin header. Verify all `__()` calls use `'ai-seo-tool'` as the domain.

7. **Build output**: Run `cd wordpress-plugin && npm run build` to ensure final build is clean.

8. **PHP syntax check**: Run `php -l` on all PHP files.
