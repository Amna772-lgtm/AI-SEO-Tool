## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), tight inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `py-2`) |
| md | 16px | Default element and form field spacing (`px-4`, `gap-4`) |
| lg | 24px | Card internal padding (`p-6` = 24px), section padding |
| xl | 32px | Auth card max width internal breathing room |
| 2xl | 48px | Vertical centering offset on auth pages |
| 3xl | 64px | Not used in this phase |

Exceptions: Input touch targets use `py-2 px-3` (8px/12px) matching existing ScheduleFormModal inputs. Auth page card uses `p-6` (24px) matching existing modal card padding.

Source: measured from `ScheduleFormModal` in `frontend/app/components/schedules/SchedulesTab.tsx` lines 118–124 and `StatusCard` in `frontend/app/page.tsx` lines 140–154.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / input text | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / helper / table cell | 12px (`text-xs`) | 400 (regular) | 1.4 |
| Card heading / modal title | 14px (`text-sm`) | 600 (semibold) | 1.2 |
| Section micro-label | 10px (`text-[10px]`) | 600 (semibold), uppercase, tracked | 1.0 |

Rules:
- Maximum 2 weights used: regular (400) and semibold (600).
- No font size below 10px.
- Monospace font (`font-mono`) used only for score numbers and status codes — same as existing codebase.
- Auth page heading ("Sign in to AI SEO Tool") uses `text-sm font-semibold` — no display-size heading. Matches the compact Screaming Frog aesthetic.

Source: `frontend/app/page.tsx` — `text-xs font-semibold uppercase tracking-wide text-[var(--muted)]` pattern (line 97), `text-sm font-semibold text-[var(--accent)]` (line 610), `text-xs` throughout table rows.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f6f8` (`--background`) | Page background behind auth card, app background |
| Secondary (30%) | `#ffffff` (`--surface`) + `#eef0f3` (`--surface-elevated`) | Auth card surface, input backgrounds, sidebar |
| Accent (10%) | `#166534` (`--accent`) | Primary CTA buttons only, active nav item, logo text |
| Destructive | `#dc2626` (`--error`) | Inline field validation errors, logout confirmation text |

Accent reserved for:
- "Sign In" submit button (auth pages)
- "Create account" submit button (signup page)
- "Sign in" re-authenticate button inside session-expired modal
- Active sidebar nav item background
- App logo/wordmark

Accent is NOT used for: links, secondary buttons, disabled states, borders, or any informational states.

Semantic colors already defined in `globals.css`:
- `--success: #15803d` — not used in this phase's auth UI
- `--warning: #ca8a04` — not used in this phase's auth UI
- `--error: #dc2626` — inline validation errors under form fields
- `--muted: #6b7280` — placeholder text, field labels, helper text

Source: `frontend/app/globals.css` — full token set lines 4–18.

## Component Inventory

New components required for this phase:

### 1. `/login` Page (`frontend/app/login/page.tsx`)

Layout: Full-page centered card. No sidebar, no top header.

```
bg-[var(--background)]  ← full page
  └── flex items-center justify-center min-h-screen
        └── card: w-full max-w-sm rounded-lg bg-[var(--surface)] border border-[var(--border)] p-6
              ├── Logo row: SVG icon + "AI SEO TOOL" wordmark (text-sm font-bold text-[var(--accent)])
              ├── Heading: "Sign in to your account" (text-sm font-semibold text-[var(--foreground)] mt-4 mb-1)
              ├── Subtext: "Enter your email and password below." (text-xs text-[var(--muted)] mb-4)
              ├── Form:
              │     ├── Email field (label + input)
              │     ├── Password field (label + input)
              │     ├── Error message area (text-xs text-[var(--error)])
              │     └── Submit button: "Sign In" (full-width, accent)
              └── Footer link: "Don't have an account? Create one" → /signup (text-xs text-[var(--muted)])
```

### 2. `/signup` Page (`frontend/app/signup/page.tsx`)

Layout: Identical card structure to `/login`. Additional Name field between Email and Password.

```
card (same dimensions as /login)
  ├── Logo row (identical)
  ├── Heading: "Create your account" (text-sm font-semibold)
  ├── Subtext: "Fill in your details to get started." (text-xs text-[var(--muted)])
  ├── Form:
  │     ├── Name field (label + input)
  │     ├── Email field (label + input)
  │     ├── Password field (label + input, type=password)
  │     ├── Error message area (text-xs text-[var(--error)])
  │     └── Submit button: "Create Account" (full-width, accent)
  └── Footer link: "Already have an account? Sign in" → /login (text-xs text-[var(--muted)])
```

### 3. Session-Expired Modal (`frontend/app/components/auth/SessionExpiredModal.tsx`)

Overlays the current page when any API call returns 401. User re-authenticates inline.

Layout: Reuses existing modal pattern from `ScheduleFormModal`.

```
fixed inset-0 z-50 flex items-center justify-center bg-black/60
  └── card: w-full max-w-sm rounded-lg bg-[var(--surface)] border border-[var(--border)] p-6
        ├── Heading: "Session expired" (text-sm font-semibold text-[var(--foreground)])
        ├── Body: "Your session has expired. Sign in again to continue." (text-xs text-[var(--muted)] mt-1 mb-4)
        ├── Email field (pre-filled with current user's email from AuthContext — read-only)
        ├── Password field (label + input, type=password)
        ├── Error message area (text-xs text-[var(--error)])
        └── Submit button: "Sign In" (full-width, accent)
```

No close/dismiss button — the modal is not dismissable. User must authenticate or navigate to /login.

### 4. Sidebar Logout Section (`frontend/app/page.tsx` — sidebar `<aside>` modification)

Added at the bottom of the sidebar `<aside>` with `mt-auto`. The sidebar currently ends at line 639 after `</nav>`.

```
aside (existing, w-48 flex flex-col)
  ├── Logo header (existing)
  ├── nav (existing nav items)
  └── mt-auto border-t border-[var(--border)] p-3   ← NEW
        └── flex items-center justify-between gap-2
              ├── User info (truncated):
              │     ├── Name: text-xs font-semibold text-[var(--foreground)] truncate
              │     └── Email: text-[10px] text-[var(--muted)] truncate
              └── Logout button:
                    icon-only SVG (log-out icon, 14x14)
                    className: text-[var(--muted)] hover:text-[var(--error)] transition-colors
                    title="Sign out"
                    aria-label="Sign out"
```

### 5. `AuthContext` + `useAuth` Hook (`frontend/app/lib/auth.tsx`)

Not a visual component. Provides `{ user, loading, signOut }` to the component tree. Wrapped in `layout.tsx`. No UI output of its own.

### 6. Form Input Pattern (used in all auth forms)

Identical to the input pattern in `ScheduleFormModal` (lines 133–138):

```tsx
<label className="mb-1 block text-xs" style={{ color: "var(--muted)" }}>
  {label}
</label>
<input
  className="w-full rounded px-3 py-2 text-xs"
  style={{ background: "var(--surface-elevated)", border: "1px solid var(--border)", color: "var(--foreground)" }}
/>
```

Focus state: `focus:outline-none focus:ring-1 focus:ring-[var(--accent)]` — 1px ring in accent color.

## Interaction States

### Form Fields

| State | Visual |
|-------|--------|
| Default | `bg-[var(--surface-elevated)]` + `border-[var(--border)]` |
| Focus | `ring-1 ring-[var(--accent)]` outline, no border color change |
| Error | `border-[var(--error)]` + inline error text below field in `text-xs text-[var(--error)]` |
| Disabled | `opacity-50 cursor-not-allowed` |

### Submit Button

| State | Visual |
|-------|--------|
| Default | `bg-[var(--accent)] text-white` |
| Hover | `bg-[var(--accent-hover)]` |
| Loading/submitting | `opacity-50 cursor-not-allowed` + button label becomes "Signing in..." or "Creating account..." |
| Disabled | `opacity-50` |

### Sidebar Logout Button

| State | Visual |
|-------|--------|
| Default | `text-[var(--muted)]` |
| Hover | `text-[var(--error)]` |

No confirmation dialog for logout — it is not a destructive data action. Clicking immediately POSTs to `/auth/logout`, clears the cookie, and redirects to `/login`.

## Page Layout: Auth Routes vs. App Shell

| Route | Layout |
|-------|--------|
| `/login` | Standalone page — no sidebar, no top header. Centered card on `--background`. |
| `/signup` | Standalone page — no sidebar, no top header. Centered card on `--background`. |
| All other routes | Existing app shell (sidebar + top header). AuthProvider wraps children in `layout.tsx`. |

The Next.js middleware redirects unauthenticated users from any non-`/login`, non-`/signup` route to `/login` before the page renders. No app shell content is ever visible to unauthenticated users.

## Accessibility Contract

- All form inputs have associated `<label>` elements (not placeholder-only).
- Error messages use `role="alert"` so screen readers announce them immediately.
- The session-expired modal traps focus within the modal card while open.
- Logout icon button includes `aria-label="Sign out"` and `title="Sign out"`.
- Auth page cards do not use `<h1>` headings (the heading is `text-sm` styled, not a semantic heading level change) — matches existing codebase pattern where modals use `<h2 className="text-sm font-semibold">`.
- Password fields use `type="password"` — no show/hide toggle in this phase.
- Tab order on login: Email → Password → Submit button → Footer link.
- Tab order on signup: Name → Email → Password → Submit button → Footer link.
