## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual TailwindCSS v4 with CSS variables |
| Preset | not applicable |
| Component library | none — hand-rolled components using Tailwind utility classes |
| Icon library | none — inline SVG (14x14px viewBox="0 0 24 24", strokeWidth 2, strokeLinecap round) |

Source: `frontend/app/globals.css`, `frontend/app/layout.tsx`, `frontend/app/dashboard/page.tsx`

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding (`px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`, `px-3 py-2` for nav items) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`), card internal spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical padding |
| 3xl | 64px | Not used in admin pages — tool is dense data UI |

Exceptions:
- Sidebar width: 192px (w-48) — matches existing dashboard sidebar
- Nav item touch target: 36px min-height (py-2 + text-sm) — matches existing pattern
- Table row height: 36px (`py-2` with `text-xs`) — matches existing crawl table pattern
- Modal max-width: 384px (`max-w-sm`) — matches SessionExpiredModal pattern
- Confirmation dialog max-width: 384px (`max-w-sm`) — consistent with modal pattern

Source: `frontend/app/dashboard/page.tsx` sidebar at line 631, `SessionExpiredModal.tsx`

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table cell | 12px (text-xs) | 400 (regular) | 1.5 |
| Label / nav item | 14px (text-sm) | 400 (regular) | 1.5 |
| Section heading / card title | 12px uppercase tracking-wide (text-xs font-semibold uppercase tracking-wide) | 600 (semibold) | 1.2 |
| Display / sidebar logo | 14px (text-sm) | 600 (semibold) | 1.2 |

Notes:
- All four roles use Geist Sans
- Monospace font (Geist Mono) used only for metric values, API key display, and domain names in tables
- 10px (`text-[10px]`) permitted for secondary user info in sidebar footer and badge labels — matches existing pattern
- No sizes outside this set: do not introduce 16px, 18px, or 20px body text in admin pages

Source: `frontend/app/dashboard/page.tsx` lines 637–680, `SessionExpiredModal.tsx`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` = #f5f6f8 | Page background, main content area behind cards |
| Secondary (30%) | `var(--surface)` = #ffffff, `var(--surface-elevated)` = #eef0f3 | Admin sidebar background, all cards and panels, table rows (hover), input backgrounds |
| Accent (10%) | `var(--accent)` = #166534 | Active nav item background in admin sidebar only, primary action buttons (Save, Apply, Confirm) |
| Destructive | `var(--error)` = #dc2626 | Delete user button, delete audit button, ban domain button — destructive actions only |

Semantic colors (not accent, not destructive — status indicators only):
- `var(--success)` = #15803d — enabled status badge, active plan badge
- `var(--warning)` = #ca8a04 — degraded state indicator, disabled user badge
- `var(--muted)` = #6b7280 — secondary labels, inactive nav text, empty state body
- `var(--border)` = #d1d5dc — all card borders, table row dividers, sidebar dividers

Accent reserved for:
1. Active admin nav item background (Dashboard / Users / System / Moderation) — filled green pill
2. Primary action buttons: "Save Changes", "Apply Override", "Confirm" in destructive dialogs (white text on accent background)
3. Admin sidebar logo text — matches existing dashboard pattern

Do NOT use accent for: status badges, metric values, chart lines, hover states on non-active nav items, informational icons.

Source: `frontend/app/globals.css`, `frontend/app/dashboard/page.tsx`

## Admin Layout Contract

### Admin Shell (`frontend/app/admin/layout.tsx`)

The admin panel uses a two-column fixed layout mirroring the existing dashboard:

```
[AdminSidebar 192px] | [Main content area flex-1 overflow-auto]
```

- Outer wrapper: `flex h-screen overflow-hidden bg-[var(--background)] text-[var(--foreground)]`
- Admin sidebar: `flex w-48 shrink-0 flex-col border-r border-[var(--border)] bg-[var(--surface)]`
- Main content: `flex-1 overflow-auto p-6`

### Admin Sidebar

Structure (top to bottom):
1. Logo bar: icon + "ADMIN" label — `border-b border-[var(--border)] px-4 py-3.5`
2. Section label: "ADMIN MENU" — `text-[10px] font-semibold uppercase tracking-widest text-[var(--muted)] mb-1 px-3`
3. Nav items: Dashboard, Users, System, Moderation — `rounded-md px-3 py-2 text-sm`
   - Active: `bg-[var(--accent)] font-medium text-white`
   - Inactive: `text-[var(--muted)] hover:bg-[var(--surface-elevated)] hover:text-[var(--foreground)]`
4. Footer: admin email + sign-out button — `mt-auto border-t border-[var(--border)] p-3`

Nav icons: 14x14px inline SVG, strokeWidth 2, strokeLinecap round (matching existing sidebar)

### Page Structure (each admin page)

Each admin page (`/admin/dashboard`, `/admin/users`, `/admin/system`, `/admin/moderation`) follows this structure:

```
Page header (32px bottom margin)
  h1: page title — text-sm font-semibold text-[var(--foreground)]
  p:  subtitle  — text-xs text-[var(--muted)]

Content area (flex-col gap-4)
  Summary cards row (grid grid-cols-2 lg:grid-cols-4 gap-3)
  Primary data section (rounded-lg border bg-[var(--surface)] p-4)
  Secondary sections below
```

## Page-Level Interaction Contracts

### `/admin/dashboard` — Analytics Dashboard

**Summary cards row (top):** 4 StatCards — Total Users, Total Audits, MRR, Active Subscriptions

**Trend section:** Two AdminTrendChart components side by side on lg screens, stacked on sm.
- Chart 1: Signup trend (last 30 days) — single accent line
- Chart 2: Audit volume trend (last 30 days) — single line (#0891b2)

**System health row:** 4 StatCards — Queue Depth, Failed Jobs, Avg Audit Duration, Redis Memory

Loading state: Each StatCard shows a placeholder `h-6 w-20 animate-pulse rounded bg-[var(--surface-elevated)]` where the value would appear. Charts show a `min-h-[160px] rounded animate-pulse bg-[var(--surface-elevated)]` skeleton.

Error state: StatCard value shows "—" in `text-[var(--muted)]` with no delta badge. No chart rendered.

### `/admin/users` — User Management

**Search + filter bar:**
```
flex gap-2 mb-4
  input: search by email/name — flex-1 rounded border border-[var(--border)] px-3 py-2 text-xs bg-[var(--surface)]
  select: filter by plan (All / Free / Pro / Agency) — same styling as input, w-32
  select: filter by status (All / Active / Disabled) — same styling, w-32
```

**User table columns:** #, Email, Name, Plan, Signup Date, Audits, Status, Actions

**Actions cell:** Two buttons in each row:
- Plan override: dropdown select (`<select>` inline in cell), onChange triggers save immediately with loading spinner replacing the select momentarily
- Disable/Enable: toggle button — "Disable" when active (border + muted text), "Enable" when disabled (border + accent text)
- Delete: trash icon button (text-[var(--muted)] hover:text-[var(--error)]) — opens ConfirmDialog

**Empty state:** No table rows. Show centered message: "No users found" in `text-sm text-[var(--muted)]` with "Try adjusting your search or filter" below in `text-xs text-[var(--muted)]`.

### `/admin/system` — System Controls

**Three sections stacked:**

**Section 1 — Celery Queue:**
```
rounded-lg border border-[var(--border)] bg-[var(--surface)] p-4
  header: "QUEUE STATUS" label + last-updated timestamp
  3 stats inline: Active tasks | Pending tasks | Worker status (Online/Offline badge)
  Refresh button: "Refresh" — rounded border px-3 py-1.5 text-xs
```
Worker offline state: "Worker Offline" badge in `bg-[#fee2e2] text-[#991b1b]` — all queue stats show "—"

**Section 2 — Feature Toggles:**
Three ToggleRows:
- Competitor Tracking (enabled/disabled)
- New User Signups (open/paused)
- Maintenance Mode (off/on)

Save confirmation: inline success message `"Saved" text-xs text-[var(--success)]` appears for 2 seconds after each toggle change, then fades.

**Section 3 — API Keys:**
Two MaskedApiKey rows — Google PSI API Key, Anthropic API Key.
Each row has an "Edit" button that replaces the masked display with an `<input type="password">` + "Save" and "Cancel" buttons.
On Save: show spinner, then either inline success or error message.

### `/admin/moderation` — Content Moderation

**Two sections side by side on lg, stacked on sm:**

**Section 1 — All Audits table:**
Columns: Domain, User Email, Date, Score, Actions (Delete button)
Search bar above: filter by domain or user email
Delete action: opens ConfirmDialog before deletion

**Section 2 — Domain Blocklist:**
```
rounded-lg border border-[var(--border)] bg-[var(--surface)] p-4
  header: "DOMAIN BLOCKLIST" label + count badge
  add form: flex gap-2
    input: domain input — flex-1 text-xs
    button: "Ban Domain" — accent bg, white text, text-xs
  list: scrollable max-h-64, each row: domain + reason + ban date + Unban button
```

Per-user rate limit overrides appear below as a separate table: User Email, Current Quota, Override Value, Actions (Edit/Remove)

## Accessibility Contract

- All action buttons have `aria-label` describing the specific action and target (e.g., `aria-label="Delete user john@example.com"`)
- ConfirmDialog uses `role="dialog"` `aria-modal="true"` `aria-labelledby` pointing to dialog heading — same as existing SessionExpiredModal
- Tables use `<thead>` with `<th scope="col">` for all columns
- Status badges use `role="status"` where they update dynamically
- Maintenance mode banner uses `role="alert"` `aria-live="polite"`
- Toggle buttons use `aria-pressed` (true/false) and `aria-label` describing the feature
- Form inputs have associated `<label>` elements (htmlFor pattern) — matches existing forms
